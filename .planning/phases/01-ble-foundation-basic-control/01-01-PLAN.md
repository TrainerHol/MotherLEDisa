Service UUID: 0000fff0-0000-1000-8000-00805f9b34fb
Characteristic UUID: 0000fff3-0000-1000-8000-00805f9b34fb
Command Start Byte: 0x7E
Command End Byte: 0xEF
Command Length: 9 bytes fixed

<!-- MELK Initialization Sequence -->
Init Command 1: 0x7e 0x07 0x83
Init Command 2: 0x7e 0x04 0x04

<!-- Power Commands -->
Power ON: 0x7e 0x04 0x04 0xf0 0x00 0x01 0xff 0x00 0xef
Power OFF: 0x7e 0x04 0x04 0x00 0x00 0x00 0xff 0x00 0xef

<!-- Color Command Format -->
Set Color: 0x7e 0x07 0x05 0x03 R G B 0x10 0xef

<!-- Brightness Command Format -->
Set Brightness: 0x7e 0x00 0x01 LEVEL 0x00 0x00 0x00 0x00 0xef (LEVEL: 0-100)

<!-- Effect Command Format -->
Set Effect: 0x7e 0x00 0x03 EFFECT_ID SPEED 0x00 0x00 0x00 0xef (SPEED: 0-100)

<!-- Device Name Prefixes for Filtering -->
Compatible: ELK-, MELK-, LEDBLE, ELK-BULB, ELK-LAMPL

**MelkProtocol.kt:** Create object with:
- SERVICE_UUID = "0000fff0-0000-1000-8000-00805f9b34fb"
- CHAR_UUID = "0000fff3-0000-1000-8000-00805f9b34fb"
- fun isCompatibleDevice(name: String?): Boolean (checks ELK-, MELK-, LEDBLE prefixes)
- fun isMelkDevice(name: String?): Boolean (checks MELK- prefix only)
- suspend fun initializeMelkDevice(peripheral, characteristic): sends 0x7e 0x07 0x83 then 0x7e 0x04 0x04 with 50ms delays
- fun powerOn(): ByteArray = 0x7e 0x04 0x04 0xf0 0x00 0x01 0xff 0x00 0xef
- fun powerOff(): ByteArray = 0x7e 0x04 0x04 0x00 0x00 0x00 0xff 0x00 0xef
- fun setColor(r, g, b): ByteArray = 0x7e 0x07 0x05 0x03 R G B 0x10 0xef
- fun setBrightness(level): ByteArray = 0x7e 0x00 0x01 LEVEL(0-100) 0x00 0x00 0x00 0x00 0xef
- fun setEffect(effectId, speed): ByteArray = 0x7e 0x00 0x03 ID SPEED 0x00 0x00 0x00 0xef

**BleCommand.kt:** Sealed class for command types:
```kotlin
sealed class BleCommand {
    abstract val data: ByteArray
    data class Power(override val data: ByteArray): BleCommand()
    data class SetColor(override val data: ByteArray): BleCommand()
    data class SetBrightness(override val data: ByteArray): BleCommand()
    data class SetEffect(override val data: ByteArray): BleCommand()
}
```

**TowerCommandQueue.kt:** Channel-based queue per RESEARCH.md Pattern 1:
- Channel<BleCommand>(UNLIMITED)
- CoroutineScope(Dispatchers.IO + SupervisorJob())
- Process one command at a time with 20ms delay between
- Use WriteType.WITHOUT_RESPONSE for speed
- Timber.e logging on failure

**ConnectionState.kt:**
```kotlin
sealed class ConnectionState {

    data class Connected(val address: String) : ConnectionState()
    data class Error(val message: String) : ConnectionState()
}
```

**DeviceScanner.kt:** Using Nordic Scanner Compat:
- BluetoothLeScannerCompat with ScanFilter for ELK-, MELK-, LEDBLE prefixes
- SCAN_MODE_LOW_LATENCY for continuous scanning (D-01)

**TowerConnectionManager.kt:** Multi-device management:

- connect(address): Uses Nordic BLE Library with 10s timeout, MTU request, service discovery, MELK init if needed
- disconnect(tower): Clean disconnect, update Room
- setColor/setBrightness/setEffect: Route through TowerCommandQueue
